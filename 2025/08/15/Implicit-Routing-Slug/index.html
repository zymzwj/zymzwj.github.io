2. What is “Implicit Routing”

The word “implicit” means you don’t need to manually write the intermediate details; the system automatically handles them for you.
In Laravel, Implicit Route Model Binding means:

As long as you specify a model class type in the route parameter, Laravel will automatically fetch the corresponding data from the database and pass you the model object.

3. Comparing with and without Implicit Routing

Without implicit routing (manual lookup)

  • {id} is the number in the URL
  • You need to manually query the database using User::findOrFail($id)

With implicit routing

  • {user} matches the route parameter name with the variable name in the function
  • (User $user) tells Laravel this is a User model
  • Laravel will automatically run User::findOrFail(parameter value) and pass the found object to you
  • If not found, Laravel will automatically return a 404

4. Advantages of Implicit Routing

  1. Less code
    No need to manually write find() or findOrFail()
  2. Automatic 404
    If no record is found, Laravel automatically returns a 404 page
  3. High readability
    Others can tell at a glance that $user is a model object, not just a number

1. Understanding “Slug” with a Real-life Example

Imagine looking for a book on a bookshelf:

  • If the label says “123”, you must check the catalog to know that 123 is The Three-Body Problem
  • If the label says “san-ti”, you immediately know it’s The Three-Body Problem

A slug is such a “meaningful label”—it uses words (instead of numeric IDs) in URLs to represent a piece of content.

3. Rules for Slugs (URL-friendly)

Slugs usually:

  • Contain only lowercase letters
  • Use - to separate words (no spaces)
  • Avoid special characters (like ?, &, #)
  • Are generally generated from the title

Examples:

Title Slug
Hello World hello-world
Laravel 隐式路由 laravel-yin-shi-lu-you
我爱PHP wo-ai-php

4. Benefits of Slugs

  1. Aesthetic: Clean, meaningful URLs
  2. Memorable: Easy for users to remember
  3. SEO-friendly: Search engines can read keywords directly from URLs
  4. Better UX: Users can guess the content from the link itself

4. How They Work Together

  1. User visits /posts/hello-world
  2. Laravel matches the route /posts/{post}
  3. Sees parameter type Post → triggers implicit model binding
  4. Laravel calls Post::getRouteKeyName()
    • If not overridden: returns 'id'
    • If overridden: returns 'slug'
  5. Laravel decides which column to query:
    • id → search by primary key
    • slug → search by slug
  6. If found, injects the model object into your controller or closure
  7. If not found, automatically returns a 404
