5. Template Inheritance (Layout Reuse)

The biggest advantage of Blade is template inheritance, which allows all pages to share the same header and footer.

What is Template Inheritance?

  • Parent layout: Holds the common structure (<html>, header, navigation, footer).
  • Placeholder (@yield): Tells child pages “this is where you fill in content.”
  • Child view: Uses @extends to inherit the parent layout and @section to fill placeholders.
  • Partial (@include): Break small pieces like navigation and footer into separate files, include them in the parent layout.
  • Page-specific additions (@stack / @push): For extra scripts/styles needed by certain pages, push them into a “stack” and output them in the parent layout.

Recommended Directory Structure

1) Parent Layout: resources/views/layouts/app.blade.php

Key points:

  • @yield('title', 'My Site'): Uses default title if the child view doesn’t provide one.
  • @stack('styles') / @stack('scripts'): Wait for child pages to @push content into them.

2) Partial: resources/views/partials/nav.blade.php

About Table Creation

In Laravel, after creating a migration file for a table, running php artisan migrate actually applies it to the database.

Two steps:

  1. Create migration file: blueprint of table structure

    Example:

    This command will generate a PHP file containing the instructions for creating or modifying a database table:

    tips:This step only writes the table structure definition, but no changes have been made in the database.

  2. Run migration: executes the blueprint in the DB

    • Laravel reads the migration files in the database/migrations folder and actually performs those schema changes in the database (e.g. CREATE TABLE users (...)).

      In other words:

    • The migration file = a blueprint for the database schema

    • The migrate` command = commits that blueprint to the database

Advantages:

  • Version control in Git
  • Same structure across environments
  • Rollback capability

✅ In short: The migration file is the “plan,” and php artisan migrate is the “construction.” Only after migrating will the table exist in the database.
