路由

1. 路由的作用

在 Laravel 里，路由就是规定：

当用户访问某个 URL 时，要返回什么内容。

比如：

  • 用户访问 /hello → 返回一句 “Hello Laravel”
  • 用户访问 /about → 返回一个 HTML 页面

Laravel 会先匹配路由，再交给控制器或直接返回内容。

5. 模板继承（布局复用）

Blade 最大的好处是 模板继承，可以让所有页面用同一个头部、底部。

(详细讲解)

###模板继承到底是啥？

  • 父布局（layout）：放通用结构（<html>、头部、导航、页脚）。
  • 占位符（@yield）：告诉子页面“这里留给你填”。
  • 子页面（view）：用 @extends 继承父布局，在 @section 里把内容塞进对应的占位符。
  • 碎片（@include）：把导航、页脚这种小块单独拆文件，父布局里按需引入。
  • 按页追加（@stack / @push）：某些页面需要额外的脚本/样式，就往“堆栈”里 @push，父布局在固定位置 @stack 输出。

推荐目录结构

resources/views/
├─ layouts/
│ └─ app.blade.php # 父布局
├─ partials/
│ ├─ nav.blade.php # 导航碎片
│ └─ footer.blade.php # 页脚碎片
├─ home.blade.php # 子页面1
└─ about.blade.php # 子页面2

1）父布局：resources/views/layouts/app.blade.php

要点：

  • @yield('title', '我的站点')：子页面不写标题就用默认值。
  • @stack('styles') / @stack('scripts')：等着子页面用 @push 往里塞。

中间件

1. 中间件是什么？

简单来说，中间件就是 请求（Request） 和 响应（Response） 之间的一道“过滤器”。
它可以在请求到达控制器之前、或者响应返回浏览器之前，对数据进行检查、修改、拦截、记录等操作。

你可以把它想象成机场安检：

  • 乘客（Request）来了 → 先经过安检（Middleware）
  • 安检通过 → 进入登机口（Controller）
  • 返程时也可能再检查行李（Response 处理）

2. 中间件的作用

Laravel 自带很多中间件，也允许你自己写。常见用途有：

场景 示例中间件 作用
认证 auth 只允许已登录用户访问某些路由
权限 can:delete-post 检查用户是否有权限执行操作
防护 VerifyCsrfToken 防止 CSRF 攻击
节流 throttle:60,1 限制 API 调用频率（每分钟 60 次）
CORS HandleCors 允许跨域访问
日志 自定义 在请求进入/离开时记录日志
数据加工 自定义 修改请求参数或统一响应格式

6. 你需要记住的核心点

  1. 位置：中间件处于 Request 和 Response 之间，像过滤器一样。
  2. 功能：可以拦截、修改、放行、拒绝请求，也可以修改返回的响应。
  3. 注册方式：全局、分组、单路由。
  4. 执行顺序：请求时顺序进入，响应时反向返回。
  5. 应用场景：认证、权限、日志、限流、安全、防爬虫、数据预处理等。

顺序是这样的：

  1. 浏览器请求
  2. 全局中间件（Kernel::$middleware）——所有请求都会经过
  3. 分组中间件（Kernel::$middlewareGroups）——如 web、api
  4. 路由中间件（Kernel::$routeMiddleware）——路由上单独声明的
  5. 控制器 Controller——执行业务逻辑
  6. 路由中间件（返回阶段）
  7. 分组中间件（返回阶段）
  8. 全局中间件（返回阶段）
  9. 浏览器接收响应
