1. First, understand what REST is

REST (Representational State Transfer) is a design style, not a specific technology or protocol.
Its core ideas are:

  • Use URLs to represent resources
  • Use HTTP verbs to represent actions
  • The server can return resources in different formats (JSON, HTML, etc.)

For example, you have a “photos” resource:

  • GET /photos → Retrieve all photos
  • POST /photos → Create a new photo
  • GET /photos/1 → Retrieve the photo with ID 1
  • PUT/PATCH /photos/1 → Update the photo with ID 1
  • DELETE /photos/1 → Delete the photo with ID 1

In this way, the URL only describes the “resource” without describing the action, and the action is determined by the HTTP method.

2. What is a RESTful framework

When a web framework (such as Laravel, Django, Spring Boot)
has built-in support for REST-style routing rules, controller method mapping, and data return formats, we can call it a RESTful framework or a framework that supports the RESTful style.

In Laravel, Route::resource() is a typical RESTful route generator:

HTTP Method URL Controller Method Purpose
GET /photos index List all photos
GET /photos/create create Show the form to create a photo
POST /photos store Save a new photo
GET /photos/{photo} show View a specific photo
GET /photos/{photo}/edit edit Show the edit form
PUT/PATCH /photos/{photo} update Update a photo
DELETE /photos/{photo} destroy Delete a photo

3. Benefits of RESTful

  • Uniformity: All resources follow the same rules, URLs are consistent, and APIs are easy to understand
  • Predictability: You can guess the function just by looking at the URL and HTTP verb
  • Better aligned with HTTP protocol: Use GET to read, POST to create, PUT to update, DELETE to remove
  • Facilitates front-end/back-end separation: Front-end can easily call RESTful APIs using Ajax / Fetch / Axios

4. How it’s reflected in Laravel

Laravel’s RESTful concept is mainly reflected in:

  • Resource routes: Route::resource() automatically generates a standard set of routes
  • Resource controllers: php artisan make:controller PostController --resource automatically generates 7 methods corresponding to REST actions
  • API resource routes: Route::apiResource() keeps only the 5 methods needed for APIs (omitting create/edit)
  • Route model binding: Automatically converts URL parameters into model objects
  • Response format: For API development, you can simply return Post::all(); to return JSON

5. One-sentence summary

REST is a resource-oriented API design concept.
A RESTful framework is a tool that helps you quickly create standardized APIs according to REST rules.
In Laravel, Route::resource() is the representative of RESTful routing.

#Restful

1. 先理解 REST 是什么

REST（Representational State Transfer，表述性状态转移）是一种设计风格，不是某种具体的技术或协议。
它的核心思想是：

  • 用 URL 表示资源（Resource）
  • 用 HTTP 动词 表示操作（行为）
  • 服务器返回的资源可以用不同格式表现（JSON、HTML 等）

比如，你有一个“照片”的资源（photos）：

  • GET /photos → 获取所有照片
  • POST /photos → 新建一张照片
  • GET /photos/1 → 获取 ID 为 1 的照片
  • PUT/PATCH /photos/1 → 更新 ID 为 1 的照片
  • DELETE /photos/1 → 删除 ID 为 1 的照片

这样，URL 只描述“资源”，不描述动作，动作由 HTTP 方法决定。

2. 什么是 RESTful 框架

当一个 Web 框架（比如 Laravel、Django、Spring Boot）
内置了对 REST 这种风格的路由规则、控制器方法映射、数据返回格式的支持，我们就可以称它是一个 RESTful 框架 或 支持 RESTful 风格的框架。

Laravel 里，Route::resource() 就是典型的 RESTful 路由生成器：

HTTP 方法 URL 控制器方法 用途
GET /photos index 列出所有照片
GET /photos/create create 显示新建照片的表单
POST /photos store 保存新照片
GET /photos/{photo} show 查看某张照片
GET /photos/{photo}/edit edit 编辑表单
PUT/PATCH /photos/{photo} update 更新照片
DELETE /photos/{photo} destroy 删除照片

3. RESTful 的好处

  • 统一性：所有资源遵循相同规则，URL 设计一致，接口易懂
  • 可预测性：只看 URL 和 HTTP 动作就能猜出功能
  • 更符合 HTTP 协议：用 GET 查、POST 增、PUT 改、DELETE 删
  • 便于前后端分离：前端用 Ajax / Fetch / Axios 调用 RESTful API 很方便

4. 在 Laravel 中的体现

Laravel 的 RESTful 思想 主要体现在：

  • 资源路由：Route::resource() 自动生成一套标准路由
  • 资源控制器：php artisan make:controller PostController --resource 自动生成 7 个方法，对应 REST 动作
  • API 资源路由：Route::apiResource() 只保留 API 需要的 5 个方法（去掉 create/edit）
  • 路由模型绑定：让 URL 参数自动转成模型对象
  • 响应格式：API 开发时可直接 return Post::all(); 返回 JSON

5. 一句话记忆

REST 是一种资源导向的 API 设计思想，
RESTful 框架 是帮你快速按照 REST 规则写出统一接口的工具。
在 Laravel 里，Route::resource() 就是 RESTful 路由的代表。
